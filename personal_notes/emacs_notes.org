** Publishing websites
- org-mode includes a publishing system that can export a _project_ to a set of HTML files ready for upload
- Configured through =org-publish-project-alist=
- The =car= of each element is a string that uniquely identifies the project
- The =cdr= of the element is the configuration for the project
  #+begin_src elisp
    (:property "value" :property "value")
  #+end_src
- Alternatively, a project may be composed of multiple sub-projects
  #+begin_src elisp
    (:components ("project-name" "project-name"))
  #+end_src
- When such a "meta-project" is published, all the individual sub-projects are processed and published in order
- Sources and destinations
  - At a minimum emacs needs to know the sources and destinations of project files
  - =:base-directory= --- specifies the source directory
  - =:publishing-directory= --- specifices the output directory
  - =:preparation-function= --- function that will be called prior to the publishing process starting
  - =:completion-function= --- function that will be called after the publishing process completes
- Selecting files
  - At a minimum, all =.org= files will be selected
  - =:base-extension= --- regular expression that matches extensions (without dot) of source files
    - If you want to get all files in the source directory, set this to =:any=
  - =:exclude= --- Regular expression that matches files to be excluded, regardless of their extension
  - =:include= --- List of files that should be included regardless of =:base-extension= and =:exclude=
  - =:recursive= --- Set this to non-nil to recursively check the base directory for files to publish

* Always load a file in its own frame
- To always load a file into its own frame, update =display-buffer-alist= to specify that emacs should load the buffer corresponding to that file in its own frame
- =display-buffer-alist= is a list of cons cells of the form =(condition . action)=
  - =condition= can be many things, but in practice it's often a regular expression that matches the buffer name
  - =action= is an alist that consists of =(functions . parameters)=
  - =functions= is a list of "action functions" applied in order --- see the documentation for =display-buffer= for a list of action functions
    - To open a buffer in a new frame, use =display-buffer-pop-up-frame=
  - =parameters= is itself an alist containing the parameters to the action function
    - For example, setting =parameters= to =((pop-up-frame-parameters . ((name . "<frame title>"))))= sets the name of the new frame used to display the given buffer
- For example, I want to load my to-do list in its own frame, so I can add the following entry to =display-buffer-alist=
  #+begin_src elisp
    ("todo\\.org" .
     ((display-buffer-pop-up-frame) .
      ((pop-up-frame-parameters . ((name . "To Do"))))))
  #+end_src

* List the major modes active in a buffer
- The major mode for a buffer is stored in the =major-mode= variable
- In order to list all the active minor modes, iterate =minor-mode-list= and check to see if the minor mode is active for the given buffer
- In order to see if a minor mode is active, check to see if its symbol is bound and non-nil in the current buffer
- Thus we can write the following function to get the list of modes:
  #+begin_src elisp
    (defun get-all-modes ()
      (interactive)
      (let ((active-modes '()))
        (mapc (lambda (mode)
                (condition-case nil
                    (when (and (boundp mode) (symbol-value mode))
                      (push mode active-modes))
                  (error nil)))
              minor-mode-list)
        (push major-mode active-modes)
        (message (string-join  (flatten-list `("Active modes:" ,(mapcar symbol-name active-modes))) " "))))
  #+end_src
