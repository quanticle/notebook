    (defun org-pmwiki-link (link desc info)
      "Transcode a LINK element into pmwiki format.
    DESC is the description part of the link, or the empty string. INFO is a plist

                                 (file-name-nondirectory path))))

                (format "[[%s | %s]]" wiki-page-name desc)
              (format "[[%s]]" wiki-page-name)))))))

* Tables

** How does PMWiki define tables?
- PMWiki defines simple tables with =||=
  - The first line of the table defines the table properties:
    #+begin_example
      || border=1
    #+end_example
  - Subsequent lines define the table contents
    #+begin_example
      || border=1
      || cell 1 || cell 2 || content 3 ||
      || row 2 cell 1 || row 2 cell 2 || row 2 cell 3 ||
    #+end_example
  - Note that there isn't a specific formatting marker to end the table
  - To create a header cell, demarcate the cell with =||!=
  - Alignment
    - To left-align contents, include no spaces preceding the content, and include a space succeeding: =||contents ||=
    - To center align contents, include a space preceding and succeeding the content: =|| contents ||=
    - To right-align contents, include a space preceding, but no space succeeding: =|| contents||=
- PMWiki defines complex tables with [[https://www.pmwiki.org/wiki/PmWiki/TableDirectives][table directives]]
  - Create a new table with =(:table :)= and close it with =(:tableend :)=
    - The =table= element can have the table-wide attributes
    - =border=
    - =width=
    - =bgcolor=
    - =cellspacing=
    - =cellpadding=
  - Header rows
    - The first cell in a header row is defined with =(:headnr :)=
    - Subsequent cells are defined with =(:head :)=
  - Data rows
    - The first cell in a new row is defined with =(:cellnr :)=
    - Subsequent cells are defined with =(:cell :)=
- I can't use table directives to define tables, because tables defined with table directives can't be nested inside lists

** How does ox-html handle tables?
- Look at these three functions:
  - =org-html-table=
  - =org-html-table-row=
  - =org-html-table-cell=
- Alignment is handled by =org-export-table-cell-alignment=

** How should ox-pmwiki handle tables?
- Very similar to how =ox-html= handles tables
- Create =org-pmwiki-table=, =org-pmwiki-table-row= and =org-pmwiki-table-cell=
- =org-pmwiki-table= will be responsible for the initial =||= with the table-wide attributes
- =org-pmwiki-table-row= will practically be a no-op
- =org-pmwiki-table-cell= will handle
  - Outputting the contents of each cell
  - Cell alignment
- =org-pmwiki-table cell= needs to know whether it is part of a header row
  - If part of a header row, output =||!= with the cell contents
  - Otherwise just output =||=
- I can use =org-export-table-cell-address= to get the a =(row . col)= (0-indexed) of the cell
- To get the total dimensions of the table, I can use =org-export-table-dimensions=

* Internal Links
- Org allows for links to other headings within the same page
- How does =ox-html= handle this?
- Let's assume, for now, that internal links are to headings only not arbitrary text
- =ox-html= assigns an ID to every headline that it exports, even if the headline isn't being referred to in a cross-reference
  - Generates IDs with =(org-html--reference headline info)=
- Uses =org-html--reference= to get the =href= for the link
  - Calls =org-element-type= to get the type of the object
  - If the type is =headline= or =inlinetask=, it calls =(org-element-property :CUSTOM_ID datum)= to get the custom ID for the headline, if it has one
  - Otherwise it looks for an =:ID= property ont the datum
  - If it has neither a =:CUSTOM_ID= nor an =:ID= property, it calls =org-export-get-reference=
- =org-html--reference= comes up with stable IDs for org objects

  - Needs to only handle headlines
  - Check if headline has a =:CUSTOM_ID= property
  - If it does, use that
  - Otherwise call =org-export-get-reference=
