#+TITLE: Alacritty

* Config
- Alacritty config file location:
  - Linux: =$HOME/.config/alacritty/alacritty.toml=
  - Windows =%APPDATA%/alacritty/alacritty.toml=
- To paste: =Ctrl-Shift-V=

* Keybindings
- Look at https://docs.rs/winit/latest/winit/keyboard/enum.NamedKey.html for the syntax for how to specify keys
- The Alacritty key binding syntax is:
  #+begin_src toml
    [keyboard]
    bindings = [{<key>, <mods>, <mode>, <command>}]
  #+end_src
  - =key= is specified as above
  - =mods= specify the modifier keys
    - =Command=, =Control=, =Option=, =Super=, =Shift=, =Alt=
    - Multiple modifiers can be combined as follows: =Control | Alt=
  - =mode= specifies the terminal mode in which the command is active
    - Specify ="AppCursor" | "AppKeypad" | "Search" | "Alt" | "Vi"= to select all modes
  - Action is the action to execute when the key or chord is pressed --- see =man 5 alacritty= for details
- Example key binding to make =F9= open a new window, in every mode except for Vi mode
  #+begin_src toml
    [keyboard]
    bindings = [{key = "F9", mods = "None", mode = "~Vi", action = "CreateNewWindow"}]
  #+end_src

* Emoji Support
- To get good emoji support on Linux, we actually need to configure =fontconfig=
- Edit =$HOME/.config/fontconfig/fonts.conf=
  #+begin_src xml
    <?xml version="1.0" ?>
    <!DOCTYPE fontconfig SYSTEM "fonts.dtd">
    <fontconfig>
      <alias>
        <family>monospace</family>
        <prefer>
          <family>Cascadia Code PL</family>
          <family>Noto Color Emoji</family>
        </prefer>
      </alias>
    </fontconfig>
  #+end_src
- Then run =fc-cache --really-force= to rebuild the fontconfig cache
- Then set the =normal=, =bold=, =italic=, and =bold italic= families to =monospace= in =alacritty.toml=
- Restart alacritty
- Emojis should now work as they do in Gnome Terminal
